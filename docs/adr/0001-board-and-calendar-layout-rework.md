# ADR-0001: 画面構成の見直し（ボード単一リスト化・完了即アーカイブ・カレンダー無限スクロール・シフト追加）

- ステータス: 採択（Accepted）
- 日付: 2026-06-16
- 関連: `task-board-spec-v1.md` v1.4（§3.3 / §3.5 / §3.8 / §5.2 / §7 / §10）

## 背景（Context）

Ichimoku の北極星は「経営者が日々の作業を**整理せずに消化していく**」こと。
運用イメージを再検討した結果、初期 UI（v1.3）に次の摩擦が見つかった。

1. **タスクボードの状態列（未着手／対応中）**: カードを置くたびに「着手中か？」の判断を強いる。
   これ自体が"整理作業"であり、ゴール（整理しない）と矛盾する。
2. **完了タスクの扱い**: 完了をグレーアウトで残し7日後にアーカイブする方式は、能動ビューに
   消化済みが残り続け、ボードのノイズになる。
3. **カレンダーの週/日ビュー＋時間グリッド**: 右カラムは約440pxと狭く、7列の時間グリッドは
   見づらい。ドラッグ移動・リサイズの実装コストも高い。
4. **勤務地（シフト）の表示要望**: 「その日はどこにいるか」（渋谷店/恵比寿店/りんご/休み等）を
   見たい。これは"用事"である予定とは性質が異なる。

## 決定（Decision）

1. **ボードはカテゴリ単位の単一リストにする（状態列を廃止）**。
   「対応中」は列ではなく**カード上の★フラグ**（内部状態 `doing`）で表す。並び順は
   `position`（fractional index）をカテゴリ単位に変更。
2. **完了＝即アーカイブ**。完了ゾーンへのドロップで `completed_at` と `archived_at` を同時に
   記録し、ボードから即消す。誤操作救済に**数秒の取り消し（undo）トースト**を出す。完了は
   **「完了履歴」side-peek** に新しい順で残り、**完了から30日で DB からも物理削除**する。
3. **カレンダーは無限スクロールのアジェンダに一本化**（週/日切替・時間グリッドを廃止）。
   時間変更は予定詳細パネル（datetime-local）で行う。
4. **シフト（勤務地）を予定とは別テーブルで持つ**。マスタ `shift_types`（名前・色・並び順）＋
   割当 `shifts`（`unique(owner_id, date)` で1日1件）。アジェンダ各日に色付きチップで表示。
   繰り返し割当は v1 では実装しない。

## 選択肢と理由（Alternatives）

- ボード: ①完全1列化 ②対応中をフラグで残す（採用）③現状維持。
  → "今やっている1件"を示す軽い焦点シグナルは残したいが、列という重い形では不要。**②**。
- 完了の30日後: ①物理削除（採用）②非表示のまま DB 保持。
  → 個人専用（RLS）かつ消化型ツールで、業務記録としての長期保持要件は確認の結果なし。
  シンプルさ優先で**物理削除**。誤操作は undo＋履歴で救済。
- シフト: ①終日予定で代用 ②専用テーブル（採用）。
  → 予定（用事）とシフト（状態）を混ぜると読めなくなる。**専用テーブル**で分離。

## 影響（Consequences）

### 良い影響

- 能動ビュー（ボード）が常に未消化だけになり、消化型のコンセプトに忠実。
- 時間グリッド（`TimeGrid`）・週/日トグルの撤去でカレンダー実装が大幅に縮小。
- シフトの色分けでスクロール時に勤務リズムが一目で読める。

### 注意・トレードオフ

- カレンダー上での**ドラッグ時間編集は廃止**（詳細パネル経由に一本化）。
- 30日**物理削除は不可逆**。undo トースト＋完了履歴で救済する設計が前提。
- 新規テーブル追加（`shift_types` / `shifts`）。マイグレーション
  `supabase/migrations/20260616000000_shifts.sql` を追加。RLS は既存3テーブルと同じ
  `auth.uid() = owner_id`。
- `tasks` の `position` 基準が（カテゴリ×状態）→（カテゴリ）に変わるため、既存データの
  並びは初回操作で再採番される（実害なし）。
- `tasks` 取得はアーカイブ込みの全件に変更（30日で物理削除＝高々30日分に収まるため許容）。

## 実装メモ

- データ層: `src/store/AppDataContext.tsx`（`tasks` から active/archived を派生・
  `completeTask`/`uncompleteTask`・30日 purge sweep・シフト CRUD）。
- ボード: `Board` / `Lane` / `BoardCell` / `TaskCard`（★フラグ）/ `CompleteZone`（履歴導線）/
  `UndoToast`。
- 完了履歴: `src/features/tasks/CompletedHistory.tsx`。
- カレンダー: `src/features/calendar/Agenda.tsx`（無限スクロール）。`TimeGrid.tsx` /
  `WeekAgenda.tsx` は削除。
- シフト: `src/features/shifts/`（`ShiftChip` / `ShiftManager` / `shiftColors`）。
