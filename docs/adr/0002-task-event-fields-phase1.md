# ADR-0002: 詳細入力項目の拡充（フェーズ1: タスク優先度・締切時刻／予定の色分け）

- ステータス: 採択（Accepted）
- 日付: 2026-06-18
- 関連: `task-board-spec-v1.md`（§3.3 タスク詳細 / §3.5 予定詳細）、`supabase/migrations/20260618000000_task_event_fields.sql`

## 背景（Context）

side-peek 詳細モーダルの再検討で、一般的なタスク管理／予定入力に対して項目の不足を確認した。
ユーザー合意のもと、段階導入することにした（フェーズ1＝本ADR、フェーズ2＝サブタスク、繰り返しは
将来の Google Calendar 連携と同時に本格実装）。

不足のうち、低コストで効果が高く単体で完結するものをフェーズ1として先行する。

## 決定（Decision）

1. **タスクに優先度（高/中/低）を追加**。`tasks.priority`（既定 `normal`、`check in ('high','normal','low')`）。
   - 詳細パネルに 3 ボタンの選択 UI。カードは既定（中）では何も出さず、**高/低のみ控えめな印**（旗アイコン＋ラベル）。
   - 高は crit 系色、低はミュート。締切の緊急度色（warn/crit）と衝突しないようアイコン併記で区別。
2. **タスクの締切に時刻を追加**。`tasks.due_time`（`time`・任意）。`due_date` がある時のみ意味を持つ。
   - 締切設定時、日付入力の隣に時刻入力（空＝日付のみ）。締切カラム・カードの締切チップ・カレンダーの
     締切表示に `M/D HH:mm` で併記。
3. **予定に色分けを追加**。`events.color`（分類色スロット `"cat-N"`・任意、null＝既定の primary アクセント）。
   - 詳細パネルに既存の `ColorPicker`（自動＋6スウォッチ）を再利用。アジェンダの予定アクセントバーに反映。

更新記録（作成/更新時刻）はユーザー判断で**残す**（タスク詳細の最下部に控えめ表示）。

## 選択肢と理由（Alternatives）

- 優先度の表現: ①明示フィールド（採用）②★フラグの多段化。
  → ★は「対応中（doing）」の意味で既に使用中。意味の二重化を避け、独立フィールドにした。
- 締切時刻: ①`due_date`＋`due_time` の分離（採用）②`due_at timestamptz` 一本化。
  → 締切カラム／カレンダーは「日」でのグルーピングが主。日付キーを維持しつつ時刻は任意付加が後方互換で安全。
- 予定の色値: ①パレットスロット参照（採用）②自由 hex。
  → 既存方針（色値は index.css に一元管理・テーマ追従・コントラスト破綻なし）に合わせる。

## 影響（Consequences）

- マイグレーション `20260618000000_task_event_fields.sql` を実 DB に適用するまでは旧スキーマのまま
  （このリモート環境に Supabase 認証情報がないため**実 DB 検証は未実施**）。既存行は既定値で後方互換。
- 「繰り返し」はフェーズ1に含めない。DB の `rrule`/`recurring_event_id`/`is_exception` 箱は既存のまま、
  実装は Google Calendar 連携（Step 12-13）と同時に行う。
- サブタスク／チェックリストはフェーズ2（`tasks.subtasks` JSONB 予定）。
