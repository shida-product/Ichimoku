# Ichimoku デザインガイド（軽量・正本）

> 目的: 回をまたいでもスタイルがブレないための「単一の正本」。
> 配色・余白・角丸の **実体は [`src/index.css`](../src/index.css)** にあり、本書はその使い方の説明に徹する。
> 見た目と操作の最終リファレンスは [`prototype-overlay.html`](../prototype-overlay.html)。

---

## 1. 原則（CLAUDE.md「引き算の美学・リニア風」を具体化）

- **線と余白で区切る／装飾で飾らない**。境界線・影は最小限。グラデーションやネオン的装飾は使わない。
- **カードUIはインタラクション要素のときだけ**。ただの表示はパネルの余白・区切り線で表現する。
- **コピーは実用語**。マーケ用語を避け、端的に（例:「タスクをパッと追加」）。
- **静かな配色**。地は Google 系ニュートラルグレー、面は白、アクセントは Google Blue に集約する。

## 2. カラートークン（セマンティック）

コンポーネントでは **生の 16 進値を書かない**。必ず下記のユーティリティを使う。
実体と対応は `src/index.css` の `:root` / `@theme inline`。

> 値は正式採用「② Google」基準。実体は `src/index.css`。

| 用途             | トークン（Tailwind ユーティリティ例）       | 値        |
| ---------------- | ------------------------------------------- | --------- |
| アプリ地         | `bg-background`                             | `#f8f9fa` |
| パネル/カード面  | `bg-card`                                   | `#ffffff` |
| くぼみ面/副次面  | `bg-secondary` / `bg-muted`                 | `#f1f3f4` |
| 既定文字         | `text-foreground`                           | `#202124` |
| 副文字           | `text-muted-foreground`                     | `#5f6368` |
| 最も淡い補助文字 | `text-ink-3`（プレースホルダ・注記）        | `#80868b` |
| 標準の線         | `border-border`                             | `#dadce0` |
| 強めの線/入力枠  | `border-input`                              | `#c0c4c9` |
| アクセント       | `bg-primary` / `text-primary` / `ring-ring` | `#1a73e8` |
| アクセント淡色   | `bg-accent` / `text-accent-foreground`      | `#e8f0fe` |

### 締切の緊急度（近日締切レーン・締切チップ）

| 緊急度         | 文字/枠                 | 地             |
| -------------- | ----------------------- | -------------- |
| 3日以内＝赤    | `text-crit`             | `bg-crit-soft` |
| 7日以内＝橙    | `text-warn`             | `bg-warn-soft` |
| それ以遠＝中立 | `text-muted-foreground` | `bg-secondary` |

### カテゴリ色（スイムレーンのドット）

カテゴリ/勤務地の色は、データには `"cat-1"`〜`"cat-6"` のスロット参照か `null`（自動）だけを保持する。
実際の色値は `src/index.css` の `--cat-1`〜`--cat-6` / `--cat-uncat` が正本。
旧データの自由 hex は `src/lib/palette.ts` が後方互換で表示するが、新規入力では増やさない。

## 3. 角丸・余白・書体

- **角丸**: 基準 `--radius = 0.5rem(8px)`。カード=`rounded-lg`、コントロール=`rounded-md`、チップ=`rounded-full`。
- **余白**: パネル内 padding は 12〜16px を基準。要素間 gap は 8〜16px。
- **書体**:
  - 本文・データ・操作系 = `--font-sans`（Noto Sans JP → system フォールバック）。数字の桁揃えが要る箇所（締切日・時刻）は `tabular` クラス。
  - 見出し・ブランド = `font-display`（`--font-display` = Zen Old Mincho 明朝）。ブランド「Ichimoku」とブロック見出し（タスクボード/カレンダー）に限定適用。
  - フォント DL を抑えるため **Zen Old Mincho はラテンのみ同梱**（各約16KB）。日本語見出しは OS の明朝（游明朝/ヒラギノ明朝）にフォールバックさせ、日本語ウェブフォント約2MB/ウェイトの読み込みを回避する。
- **タップ標的**: クリック/タップ要素は最低 24×24px（推奨 44×44px）。shadcn `Button` の既定サイズで担保。

## 4. ベースレイアウト（3カラム）

トップバー＋**3カラム（近日締切｜タスクボード｜カレンダー）**を 1 画面に固定配置する（[`AppShell`](../src/components/layout/AppShell.tsx)）。

- **近日締切は左カラム**に縦並びで常時一覧（横スクロール帯は廃止）。締切順・クリックで詳細・完了ゾーンへドラッグで消化。
- カレンダー（アジェンダ）は快適幅で上限固定し、余った横幅はボードへ回す（列が増える）。
- 各カラムは縦に伸びて高さが揃う。`lg` 未満では縦積みにフォールバック。
- **設定ボタンは効く列のヘッダーに置く**: カテゴリ管理＝タスクボードのヘッダー、勤務地管理＝カレンダーのヘッダー。全体ヘッダーはブランド名とアカウントのみ。

### カレンダー: 予定とタスク締切の描き分け（色は種別から自動）

手動の色分けは持たない（[ADR-0002](adr/0002-task-event-fields-phase1.md)「整理を生まない原則」）。種別で見た目を変える:

- **予定** = 時計アイコン＋平たい行（時刻つきは `text-primary`、終日は `text-muted-foreground`）。
- **タスク締切** = 旗アイコン＋**緊急度色の淡い塗りカード**（`bg-{crit,warn}-soft` ＋ `border-{…}/30`）。タイトル文頭は予定行と揃える。
- 勤務地チップは各日の**日付の真下**。チップ自体を色付けし、左の色ドットは置かない。

## 4-1. オーバーレイ操作モデル（§3.7・最重要）

ベース画面（締切｜ボード｜カレンダーの 3 カラム）は**常に固定**。入力・編集は重ねて行う。

| 重さ | 種類                      | 実装                                                               | 用途                                |
| ---- | ------------------------- | ------------------------------------------------------------------ | ----------------------------------- |
| 最軽 | その場編集                | 通常の input（オーバーレイなし）                                   | タイトルのインライン編集            |
| 軽   | ポップ                    | [`AnchoredPopover`](../src/components/overlay/AnchoredPopover.tsx) | クイック追加・＋予定・日付/状態選択 |
| 中   | スライドイン（side-peek） | [`SidePeek`](../src/components/overlay/SidePeek.tsx)               | タスク詳細・カテゴリ管理            |
| 封印 | ~~中央モーダル~~          | 使わない（破壊的確認のみ例外）                                     | —                                   |

**散漫でも壊れないためのルール（実装の不変条件）:**

1. **自動保存**: 保存ボタンを置かない。変更は即コミット（楽観的更新）。
   - 例外: 予定（event）は誤作成を防ぐため**保存ボタン式**（option B）。この場合も操作ボタン（削除・キャンセル・保存）は**入力項目の直下**に置く（最下部固定＝埋もれて見づらいを避ける）。
2. **クリック外／Esc で即閉じる**: 「保存しますか？」を出さない。
3. **オーバーレイは常に 1 枚**: 開く状態は呼び出し側で一元管理し、新しく開く＝前を閉じる。
   （実装例: `AppShell` の `ActiveOverlay` ステート）
4. **ドラッグ優先**: 動かせるもの（状態・カテゴリ変更）はパネルを出さず直接操作する。

## 5. やってはいけないこと（統一を壊す典型）

- コンポーネントに `#xxxxxx` や `zinc-950` 等の生 Tailwind カラーを直書きする → 必ずセマンティックトークン経由。
- 新しい中央モーダルを足す → side-peek かポップに置き換える。
- 保存ボタンを足す → 自動保存にする。
- 1 画面内に複数オーバーレイを同時に開く → 1 枚に統一する。
