# Ichimoku デザインガイド（軽量・正本）

> 目的: 回をまたいでもスタイルがブレないための「単一の正本」。
> 配色・余白・角丸の **実体は [`src/index.css`](../src/index.css)** にあり、本書はその使い方の説明に徹する。
> 見た目と操作の最終リファレンスは [`prototype-overlay.html`](../prototype-overlay.html)。

---

## 1. 原則（CLAUDE.md「引き算の美学・リニア風」を具体化）

- **線と余白で区切る／装飾で飾らない**。境界線・影は最小限。グラデーションやネオン的装飾は使わない。
- **カードUIはインタラクション要素のときだけ**。ただの表示はパネルの余白・区切り線で表現する。
- **コピーは実用語**。マーケ用語を避け、端的に（例:「タスクをパッと追加」）。
- **静かな配色**。地はオフホワイト、文字は濃いインク、アクセントは深緑 1 色に集約する。

---

## 2. カラートークン（セマンティック）

コンポーネントでは **生の 16 進値を書かない**。必ず下記のユーティリティを使う。
実体と対応は `src/index.css` の `:root` / `@theme inline`。

| 用途             | トークン（Tailwind ユーティリティ例）       | 値        |
| ---------------- | ------------------------------------------- | --------- |
| アプリ地         | `bg-background`                             | `#eef1f0` |
| パネル/カード面  | `bg-card`                                   | `#ffffff` |
| くぼみ面/副次面  | `bg-secondary` / `bg-muted`                 | `#f5f7f6` |
| 既定文字         | `text-foreground`                           | `#1c2826` |
| 副文字           | `text-muted-foreground`                     | `#5c6a66` |
| 最も淡い補助文字 | `text-ink-3`（プレースホルダ・注記）        | `#8c9794` |
| 標準の線         | `border-border`                             | `#e0e6e3` |
| 強めの線/入力枠  | `border-input`                              | `#cdd6d2` |
| アクセント       | `bg-primary` / `text-primary` / `ring-ring` | `#2f6f62` |
| アクセント淡色   | `bg-accent` / `text-accent-foreground`      | `#e4efeb` |

### 締切の緊急度（近日締切レーン・締切チップ）

| 緊急度         | 文字/枠                 | 地             |
| -------------- | ----------------------- | -------------- |
| 3日以内＝赤    | `text-crit`             | `bg-crit-soft` |
| 7日以内＝橙    | `text-warn`             | `bg-warn-soft` |
| それ以遠＝中立 | `text-muted-foreground` | `bg-secondary` |

### カテゴリ標準色（スイムレーンの帯・カード左端）

`bg-cat-jimu`(事務) / `bg-cat-keiei`(経営) / `bg-cat-saiyo`(採用) / `bg-cat-mibun`(未分類)。
※ ユーザー定義カテゴリは将来 `categories.color` を優先。未設定時にこの標準色へフォールバックする。

## 3. 角丸・余白・書体

- **角丸**: 基準 `--radius = 0.7rem(≒11px)`。カード=`rounded-lg`、コントロール=`rounded-md`、チップ=`rounded-full`。
- **余白**: パネル内 padding は 12〜16px を基準。要素間 gap は 8〜16px。
- **書体**: `--font-sans`（Noto Sans JP → system フォールバック）。数字の桁揃えが要る箇所（締切日・時刻）は `tabular` クラス。
- **タップ標的**: クリック/タップ要素は最低 24×24px（推奨 44×44px）。shadcn `Button` の既定サイズで担保。

## 4. オーバーレイ操作モデル（§3.7・最重要）

ベース画面（締切レーン＋ボード＋カレンダー）は**常に固定**。入力・編集は重ねて行う。

| 重さ | 種類                      | 実装                                                               | 用途                                |
| ---- | ------------------------- | ------------------------------------------------------------------ | ----------------------------------- |
| 最軽 | その場編集                | 通常の input（オーバーレイなし）                                   | タイトルのインライン編集            |
| 軽   | ポップ                    | [`AnchoredPopover`](../src/components/overlay/AnchoredPopover.tsx) | クイック追加・＋予定・日付/状態選択 |
| 中   | スライドイン（side-peek） | [`SidePeek`](../src/components/overlay/SidePeek.tsx)               | タスク詳細・カテゴリ管理            |
| 封印 | ~~中央モーダル~~          | 使わない（破壊的確認のみ例外）                                     | —                                   |

**散漫でも壊れないためのルール（実装の不変条件）:**

1. **自動保存**: 保存ボタンを置かない。変更は即コミット（楽観的更新）。
2. **クリック外／Esc で即閉じる**: 「保存しますか？」を出さない。
3. **オーバーレイは常に 1 枚**: 開く状態は呼び出し側で一元管理し、新しく開く＝前を閉じる。
   （実装例: `AppShell` の `ActiveOverlay` ステート）
4. **ドラッグ優先**: 動かせるもの（状態・カテゴリ変更）はパネルを出さず直接操作する。

---

## 5. やってはいけないこと（統一を壊す典型）

- コンポーネントに `#xxxxxx` や `zinc-950` 等の生 Tailwind カラーを直書きする → 必ずセマンティックトークン経由。
- 新しい中央モーダルを足す → side-peek かポップに置き換える。
- 保存ボタンを足す → 自動保存にする。
- 1 画面内に複数オーバーレイを同時に開く → 1 枚に統一する。
